Task 2.8: Protect APIs with JWT 
Public               | APIs Protected APIs 
GET /api/health      | GET /api/trainees 
POST /api/auth/login | GET /api/trainees/{id} 
                     | POST /api/trainees 
                     | PUT /api/trainees/{id} 
                     | DELETE /api/trainees/{id} 
• No token should return 401 Unauthorized. 
• Invalid token should return 401 Unauthorized. 
• Valid token should allow the API call. 
• Clients should send Authorization: Bearer <token>. 

Task 2.9: Add Pagination, Search, and Status Filter 
GET /api/trainees?pageNumber=1&pageSize=10&search=amit&status=Active 
{ 
    "pageNumber": 1, 
    "pageSize": 10, 
    "totalRecords": 25, 
    "data": [] 
}
• Use pageNumber and pageSize query parameters. 
• Support search and status filter together with pagination. 
• Do not return large datasets blindly. Use Skip() and Take() appropriately.

Task 2.11: Add Structured Logging 
    • Log user login success/failure. 
    • Log trainee created, updated, and deleted events. 
    • Log record-not-found cases and unexpected exceptions. 
    • Do not log passwords, JWT tokens, or sensitive data.
