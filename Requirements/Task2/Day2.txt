Task 2.5: Implement Password Hashing 
    • Passwords must never be stored as plain text. 
    • Store only PasswordHash in the Users table. 
    • Verify the password during login using password verification logic. 
    • Do not log passwords or password hashes. 

Task 2.6: Create Login API 
    POST /api/auth/login 
Request: 
{ 
    "username": "admin", 
    "password": "Admin@123" 
} 
Response: 
{ 
    "token": "jwt-token-value", 
    "expiresIn": 3600, 
    "user": { 
        "id": 1, 
        "username": "admin", 
        "role": "Admin" 
    } 
} 

Task 2.7: Add JWT Configuration 
    • JWT settings should come from configuration. 
    • The signing key should not be hardcoded inside controller code. Token should include useful claims: UserId, Username, Role. 
{ 
    "Jwt": { 
        "Issuer": "TraineeManagementApi", 
        "Audience": "TraineeManagementClient", 
        "ExpiryMinutes": 60 
    }
}
